## 开源，高性能，高并发，高可靠，www服务和代理服务软件


## 应用场景
	一、作为web服务软件
        1、Nginx是一个静态的web服务软件
        2、支持动静分离(静态服务自己解析，动态的服务就抛给对应服务解析)
        
    二、支持动态web服务扩展
            PHP(fastcgi_pass)
            JAVA(proxy_pass)
            Python(uwsgi_pass)	
            例：nginx结合uwsgi_pass，将程序抛给python  
        
	三、作为反向代理或负载均衡服务软件
		1、反向代理于负载均衡区别
			正向代理：由内向外，代替局域网pc，请求外部应用服务
				假如100m的带宽，可能需要20m就够了，应为数据会缓存在代理服务器
			反向代理：由外向内，代替外部的用户请求内部的应用服务器
			负载均衡：后抛请求，返回请求就不管了，而代理来回都要管
		
		2、负载均衡同类软件
			haproxy，lvs，硬件:F5，netscaler
			
		3、nginx，haproxy，lvs区别
		
	
	四、缓存服务器
		在web缓存服务方面，nginx可通过自身的proxy_cache模块实现类Squid等专业缓存软件的功能
		常见的缓存软件
			squid，varnish，nginx，ats
			
	五、支持安全的web服务(https)

## 1、创建服务脚本
[root@localhost ~]# vim /usr/lib/systemd/system/nginx.service
[Unit]
Description=nginx - high performance web server
Documentation=http://nginx.org/en/docs/
After=network.target remote-fs.target nss-lookup.target
 
[Service]
Type=forking
PIDFile=/usr/local/webserver/nginx/nginx.pid
ExecStartPre=/usr/local/webserver/nginx/sbin/nginx -t -c /usr/local/webserver/nginx/conf/nginx.conf
ExecStart=/usr/local/webserver/nginx/sbin/nginx -c /usr/local/webserver/nginx/conf/nginx.conf
ExecReload=/bin/kill -s HUP $MAINPID
ExecStop=/bin/kill -s QUIT $MAINPID
PrivateTmp=true
 
[Install]
WantedBy=multi-user.target


## 2、重新加载服务
systemctl daemon-reload

## 3、启动服务
systemctl start nginx.service
	
## 4、开机启动
systemctl enable nginx.service

ps: 遇到systemctl start nginx.service卡住问题
修改nginx.conf，设置pid路径，路径自己定义即可。

3.2 nginx工作流程

1
2
3

首先请求来了，/sbin/nginx跑起来之后，会开启一个master的主进程，该主进程去读nginx.config文件并校验有无错误，没有错误就生成多个worker进程，worker进程会读取nginx.conf配置文件来进行请求转发.

主进程master负责协调子进程(worker),如果nginx.config更新了，master会通知worker完成当前任务后结束，然后生成新的worker进程并读取nginx.config配置文件.

##  servername的多种配置
	1、可以在同一servername中匹配多个域名    server_name  test.com,test2.com;
    2、通配符匹配     server_name  *.test.com;  
    	ps：前提是能域名解析，例如我拿asd.test.com，我要能解析它的地址才会被该通配符匹配
    3、通配符结束匹配     server_name   www.test.*;
    4、正则匹配    server_name   ~^[0-9]+\.test\.com$;

## 可以隐藏后端服务器的真实地址
	例：http://192.168.101.42/index.jsp?pageNum=2
		隐藏为：http://192.168.101.42/2.html


# 1 flag标记说明：
	last	本地规则匹配完成后，继续向下匹配新的location url规则
	break	本条规则匹配完成后终止，不在匹配下面的规则
	redirect	返回302临时重定向，浏览器会显示跳转后真实的url地址
	permanent	返回301永久重定向，浏览器会显示跳转后真实的url地址
	
	ps：例如 2.html给它跳转到/index.jsp?pageNum=2，则浏览器就显示跳转后/index.jsp?pageNum=2的地址
		


# 2 配置
# ^$:这两个符号中间写正则，最后在后边写想要转变的地址
[root@localhost /]# vim /usr/local/webserver/nginx/conf/nginx.conf
location / {
			# 表示访问2.html给它跳转到/index.jsp?pageNum=2
			# rewrite ^/2.html$	/index.jsp?pageNum=2  break;
			
			# $1表示第一个匹配的数字
			rewrite ^/[0-9]+.html$	/index.jsp?pageNum=$1  break;
            root   html;
            index  index.html index.htm;
        }
        
        
# 3 重载配置
[root@localhost ~]# systemctl reload nginx


# 4 测试
http://192.168.101.42/2.html
http://192.168.101.42/66546.html

## 负载均衡+URLRewrite+反向代理
	
	
# 1、tomcat服务器开启防火墙添加信任ip(模拟内网环境)
[root@localhost ~]# systemctl start firewalld.service
[root@localhost ~]# firewall-cmd --permanent --add-rich-rule="rule family="ipv4" source address="192.168.101.42/32" port protocol="tcp" port="80" accept"
[root@localhost ~]# firewall-cmd --reload



# 2、nginx服务访问内网tomcat
[root@localhost ~]# curl http://192.168.101.186
192.168.101.186-tomcat


# 3、nginx配置负载均衡+反向代理+URLRewrite

[root@localhost /]# vim /usr/local/webserver/nginx/conf/nginx.conf
upstream fuzaijunhen {        ##  负载均衡的服务器列表
	server 192.168.101.187:80;
	server 192.168.101.188:80;
}
location / {
	rewrite ^/[0-9]+.html$	/test.html  redirect;
    proxy_pass http://fuzaijunhen
}


        
# 4 重载配置
[root@localhost ~]# systemctl reload nginx


# 4、测试

http://192.168.101.186/  没有代理访问不了

# 防止其他服务器来引用本地服务器的资源

# 客户端 --》 服务器index --》二次访问image，css等资源

# 例如我请求一个服务器的首页，这个时候是没有referers的，当二次请求index里面内链的其它资源的时候这个时候会有referers，referers会记录这个请求从哪一个页面来(第一次请求的完整域名)。
所以我二次访问的时候会判断它的referers是不是我当前服务器的域名，如果是就可以访问，如果不是就禁止访问

# 字段
	- none: 如果Header中的Referer为空，允许访问
	- blocked:在Header中的Referer不为空，但是该值被防火墙或代理进行伪装过，如不带"http://" 、"https://"等协议头的资源允许访问。
	- server_names,设置一个域或多个url，检测Referer头域值是否是这些url中的某一个



# 防盗链需要再location中配置
[root@localhost ~]# vim /usr/local/webserver/nginx/conf/nginx.conf
location ~*/(www|image) {
			# 该处表示如果检测referers为空，或者为192.168.101.42的时候允许访问
            valid_referers none 192.168.101.42; # ps：这里的地址应该是域名
            if ($invalid_referer){
            return 403;
            }
            root   /data/;
            index  index.html index.htm;
        }

## 通过keepalived的vip实现，将vip绑定到两台nginx服务器上，服务器分为一主一备，主的挂掉了vip自动绑定到备的服务器上

## 用法，用过脚本检测进程是否存活，如果没有存活把keepalived进程杀死，则该vip自动漂移到另一台服务器上


## 配置文件
# server1
global_defs{
	router_id lb111
}
vrrp_instance atguigu{
	state MASTER
	interface ens33		# 绑定的网卡
	virtual_router_id 51
	priority 100	# 选举优先级
	advert_int 1
	authentication(
		auth_type PASS
		auth_pass 1111
	)
	virtual_ipaddress{	# vip的地址，可以配置多个
		192.168.101.200
	}
}



# server2
global_defs{
	router_id lb110
}
vrrp_instance atguigu{
	state BACKUP
	interface ens33		# 绑定的网卡
	virtual_router_id 51
	priority 50	# 选举优先级
	advert_int 1
	authentication(
		auth_type PASS
		auth_pass 1111
	)
	virtual_ipaddress{	# vip的地址，可以配置多个
		192.168.101.200
	}
}
